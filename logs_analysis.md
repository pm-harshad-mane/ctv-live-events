## Detailed Observations

### Live discovery is returning data, but the top-line scores are underpowered

A fresh successful soccer live-discovery response returned:

- `excitement.aggregate_score = 0`
- `excitement.level = "medium"`
- `current_excitement = 50`
- `recent_excitement = 50`
- `expected_remaining_excitement = 50`
- `criticality.score = 80`

That combination is not internally coherent.

If:

- aggregate excitement is `0`
- but the level is `medium`
- and all component excitement values are `50`

then the scoring system is not calibrated enough yet.

The same issue appears in baseball:

- `aggregate_score = 0`
- `level = "low"`
- `current_excitement = 0`
- `recent_excitement = 0`
- `expected_remaining_excitement = 50`

This suggests:

- the prompt is producing valid shape
- but not valid score semantics

### Live active-player output is still thin

In the post-fix sample:

- some live responses had only `2` active players

That means the current prompt is still not reliably producing the richer player-level context we want.

For programmatic consumers, `active_players` should help answer:

- who is driving the game now

- which star or high-impact players are materially affecting the state

The current outputs are still too sparse for that.

That is an important shift.

## Recommended Next Prompt / Schema Changes

### 1. Add score calibration rules

The prompt should explicitly require:

- aggregate score must be consistent with component scores
- `level` must match the numeric range
- zero or near-zero aggregates must not coexist with medium/high descriptions unless justified

Examples:

- if `aggregate_score <= 20`, `level` should usually be `low`
- if `criticality.score >= 80`, reason codes must indicate strong stakes or late leverage

### 2. Add machine-friendly audience-interest scores for upcoming matches

The current `projected_competitiveness` field is not enough.

We should add:

- `audience_interest_score`
- `stakes_score`
- `star_power_score`
- `volatility_score`
- `upset_potential_score`
- `likely_closeness_score`
- `narrative_strength_score`

And ideally:

- reason codes for each

### 3. Add machine-friendly watchability scores for live matches

We should add:

- `watchability_now_score`
- `tension_score`
- `scoring_imminence_score`
- `swing_potential_score`
- `state_clarity_score`
- `evidence_strength_score`

These would be much more useful for:

- feeds
- alerts
- ranking
- audience routing

than the current summary-heavy output.

### 4. Strengthen active-player generation

The prompt should explicitly require:

- currently active players only
- at least a minimum number where the sport supports it
- player impact tied to the current live state, not generic importance

Examples:

- soccer: attacker under pressure, goalkeeper, current set-piece taker
- baseball: pitcher, batter, baserunners
- basketball: ball-handler, leading scorer, rim protector

### 5. Add score-component decomposition

Instead of:

- one top-line score with light explanation

we should prefer:

- top-line score
- component scores
- reason codes

Example:

- `audience_interest_score = 82`
- components:
  - `stakes = 90`
  - `star_power = 78`
  - `volatility = 70`
  - `rivalry = 55`
  - `certainty = 85`

That would make the output much more valuable for programmatic consumers.

### 6. Make upcoming prompt outputs less templated

The logs suggest the model is falling into a stable template:

- 3 watch reasons
- basic probability pair
- one competitiveness number

We should explicitly ask for:

- more differentiated reasoning
- component scores
- stronger evidence-based distinctions between matches

## Updated Bottom Line

The post-fix OpenAI sample shows:

- the **live-discovery parse-failure issue is substantially improved**
- **upcoming responses are now the main reliability gap**
- **score quality and machine-usable scoring depth are now the main product gap**

So the next prompt iteration should focus less on transport survival and more on:

- richer structured scoring
- score calibration
- stronger active-player utility
- upcoming-match audience-interest modeling
